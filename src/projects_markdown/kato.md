Kato is a proof-of-concept notation language — an alternative to YAML and JSON — designed to be both human and agent friendly. It aims to combine JSON's predictability, TOML's explicitness, and YAML's readability, with first-class tooling and agent metadata baked in. The name comes from the Mauritian Creole *"kato ver"* (green box).

The big rule: one obvious AST. No indentation semantics, no implicit dates, no surprise booleans, no anchors, no hidden merge behaviour, no magic interpolation. The repo contains the evolving spec, sample `.kato` files, and reference parsers in Go and TypeScript.
